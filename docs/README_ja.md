# Henry - AI 越境ECエキスパート対話システム

[English](../README.md) | **日本語** | [简体中文](./README_zh.md)

ローカルLLMを活用したインテリジェントなEコマース知識ベース応答システムです。RAG（検索拡張生成）技術を用いて、越境ECに関する質問に回答します。

## 主な機能

- 🤖 **マルチモデル対応**: OLLAMA (ローカル), OpenAI, Anthropic など、複数のLLMプロバイダーをサポート
- 📚 **知識ベース管理**: 越境EC関連の記事（TikTok、AIツール、財務など）を自動的にインデックス化
- 🔍 **ハイブリッド検索**: セマンティック検索とキーワードマッチングを組み合わせ、精度の高い回答を実現
- 💬 **マルチターン対話**: 文脈を保持し、連続した質問に対応
- 📖 **リファレンス引用**: 回答の根拠となるソース記事を明示
- 🌊 **ストリーミング応答**: ChatGPTのようなリアルタイムな回答表示
- 🎨 **高度な Markdown**: テーブル、リスト、コードのシンタックスハイライトに対応
- ⚙️ **Web設定インターフェース**: コードや環境変数を編集することなく、ブラウザ上でモデル設定や知識ベースのパスを変更可能

## 技術スタック

**バックエンド:**
- **Python 3.9+**
- **FastAPI**: 高性能なREST APIとSSEストリーミング
- **LlamaIndex**: RAGフレームワーク
- **Qdrant**: ベクトルデータベース
- **SQLite**: 対話履歴の保存
- **HuggingFace Embeddings**: 多言語対応モデル

**フロントエンド:**
- **Vanilla JavaScript**: フレームワークなしの軽量実装
- **HTML5 + CSS3**: モダンでレスポンシブなデザイン
- **Marked.js & Highlight.js**: Markdownレンダリングとコードハイライト

## クイックスタート

### 前提条件

- **Python 3.9+**
- **OLLAMA** または **oMLX** (ローカル LLM 推論用)
- **知識ベース**: Henry に学習させたい `.md` または `.xlsx` ファイルが含まれるフォルダ。

### 1. インストール

リポジトリをクローンし、必要な依存関係をインストールします：

```bash
git clone <repository-url>
cd gsd_test
pip install -r requirements.txt
```

### 2. ローカル LLM サービスの起動

ローカルモデルを使用する場合は、推論エンジン（OLLAMA または oMLX など）を起動します：

```bash
# oMLX の例
/Applications/oMLX.app/Contents/MacOS/omlx-cli launch
```

### 3. Henry の起動

起動スクリプトを実行して FastAPI サーバーを起動します：

```bash
./start_server.sh
```

サーバーは **http://localhost:8080** で利用可能になります。

### 4. セットアップと設定

Web インターフェースが開いたら：
1. サイドバー上部の**ギアアイコン (⚙️)** をクリックします。
2. **プロバイダー (Provider)**: ローカルの場合は `OLLAMA / oMLX`、クラウドモデルの場合は `OpenAI` または `Anthropic` を選択します。
3. **モデル名 (Model)**: モデル文字列を入力します（例: `Qwen3-Coder-30B-A3B-Instruct-4bit`）。
4. **ベース URL (Base URL)**: ローカルの oMLX の場合、`http://127.0.0.1:8000` を使用します。
5. **知識ベースのパス (Wiki Path)**: ドキュメントフォルダの**絶対パス**を入力します。
6. **保存**: 「保存配置 (Save Configuration)」をクリックします。システムは自動的にベクトルストアを初期化し、ドキュメントのインデックスを作成します。

### 5. チャット開始！

これでドキュメントについて質問できるようになりました。Henry は引用付きの回答を美しい Markdown 形式で提供します。

```
│   ├── search/           # 検索エンジンロジック
│   ├── rag/              # RAG コアプロセス
│   └── llm/              # LLM クライアント統合
├── frontend/             # フロントエンド静的ファイル
├── data/                 # データベースと設定ファイル
└── tests/                # テスト
```

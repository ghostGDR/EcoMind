<task type="auto">
  <name>Task 2: Configure FastAPI to serve static frontend files</name>
  <files>src/api/main.py</files>
  <action>
Update FastAPI application to serve frontend static files.

**Modifications to src/api/main.py:**

1. Add import at top:
```python
from fastapi.staticfiles import StaticFiles
```

2. After including all routers (after `app.include_router(documents.router)`), add static file mount:
```python
# Mount frontend static files
# This must be LAST - catch-all route
app.mount("/", StaticFiles(directory="frontend", html=True), name="frontend")
```

**Important:** The mount must be AFTER all API routes to avoid conflicts. StaticFiles with `html=True` enables SPA-style routing (serves index.html for non-file paths).

**Why this order:**
- API routes registered first (specific paths like /api/*)
- Static files mounted last (catch-all for /)
- FastAPI matches routes in registration order
- This ensures /api/* routes work, and / serves index.html
  </action>

| Boundary | Description |
|----------|-------------|
| Browser → FastAPI | User accesses frontend via browser, served by FastAPI |
| Frontend → API | JavaScript makes requests to /api/* endpoints |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Spoofing | Static file serving | accept | Single-user local deployment, no authentication needed in v1 |
| T-07-02 | Tampering | HTML/CSS/JS files | accept | Local filesystem access required to tamper, same as API code |
| T-07-03 | Information Disclosure | Frontend source code | accept | No secrets in frontend code, API keys stay in backend |
| T-07-04 | Denial of Service | Static file requests | accept | Local deployment, no external attackers |
| T-07-05 | Elevation of Privilege | Frontend access | accept | No privilege levels in single-user app |

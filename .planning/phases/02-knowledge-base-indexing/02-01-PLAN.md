Create `DocumentIndexer` class that orchestrates the document ingestion pipeline:

**Class structure:**
```python
class DocumentIndexer:
    def __init__(self, document_store: DocumentStore, vector_store: HenryVectorStore)
    def index_all_documents(self) -> VectorStoreIndex
    def get_index_stats(self) -> dict
```

**Implementation details:**

1. **index_all_documents()** method:
   - Call `document_store.load_all_documents()` to get List[Document]
   - Enrich each document's metadata with file path, type (markdown/excel), size
   - Call `vector_store.create_index(documents)` to create vector index
   - Return the VectorStoreIndex object
   - Handle empty document list gracefully (return None or raise clear error)

2. **get_index_stats()** method:
   - Call `vector_store.get_collection_info()` to get Qdrant collection stats
   - Return dict with: total_documents, points_count, collection_status
   - Use for verification that indexing succeeded

3. **Error handling:**
   - Catch and re-raise exceptions from DocumentStore with context
   - Catch and re-raise exceptions from VectorStore with context
   - Log document count before indexing starts

**Why this design:**
- Thin orchestration layer - delegates to existing Phase 1 components
- No business logic duplication - DocumentStore handles loading, VectorStore handles indexing
- Testable - can mock DocumentStore and VectorStore
- Metadata enrichment happens here (single responsibility)

**Chunking strategy:**
- Already configured in Phase 1: `Settings.chunk_size = 512` (in HenryVectorStore.__init__)
- LlamaIndex automatically chunks documents during `create_index()`
- Chinese text support: sentence-transformers/all-MiniLM-L6-v2 model handles Chinese
- No custom chunking logic needed - LlamaIndex's default semantic chunking is sufficient

**Excel file handling:**
- LlamaIndex SimpleDirectoryReader (used by DocumentStore) supports Excel via openpyxl
- Excel content is extracted as text automatically
- No special handling needed - treat same as Markdown

Create executable script `index_wiki.py` that indexes all documents from the user's wiki directory:

**Script structure:**
```python
#!/usr/bin/env python3
"""Index all documents from wiki directory into Qdrant vector database"""

from src.storage.document_store import DocumentStore
from src.storage.vector_store import HenryVectorStore
from src.indexing.document_indexer import DocumentIndexer

def main():
    # Initialize components
    doc_store = DocumentStore.from_env()  # Uses WIKI_PATH env var or default
    vector_store = HenryVectorStore()
    indexer = DocumentIndexer(doc_store, vector_store)
    
    # Get document count before indexing
    doc_count = doc_store.get_document_count()
    print(f"Found {doc_count['total']} documents ({doc_count['markdown']} markdown, {doc_count['excel']} excel)")
    
    # Index all documents
    print("Indexing documents...")
    index = indexer.index_all_documents()
    
    # Verify indexing succeeded
    stats = indexer.get_index_stats()
    print(f"Indexing complete: {stats['points_count']} vectors stored")
    print(f"Collection status: {stats['collection_status']}")
    
    # Clean up
    vector_store.close()

if __name__ == "__main__":
    main()
```

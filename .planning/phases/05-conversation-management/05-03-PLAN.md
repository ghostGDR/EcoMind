```python
def list_conversations(self) -> List[Dict[str, Any]]:
    """
    List all conversations with metadata
    
    Returns:
        List of dicts with: {id, title, created_at, message_count}
        Ordered by created_at DESC (newest first)
    """
    conversations = self.conversation_store.list_conversations()
    
    # Transform to frontend-friendly format
    return [
        {
            'id': conv.id,
            'title': conv.title,
            'created_at': conv.created_at,
            'message_count': len(conv.messages)
        }
        for conv in sorted(conversations, key=lambda c: c.created_at, reverse=True)
    ]
```

**2. Enhance get_history() (already exists from Plan 01):**

Plan 01 already implements get_history() which wraps conversation_store.get_conversation(). Verify it returns:

**3. Design decisions:**
- list_conversations() returns summary (not full messages) - reduces payload for UI list view
- Include message_count - helps users identify active vs empty conversations
- Sort by created_at DESC - most recent conversations first (common UX pattern)
- get_history() returns full Conversation object - detailed view for specific conversation
- Use existing ConversationStore methods - no new database queries needed

**4. Frontend integration notes:**

Create ConversationManager class that wraps QueryEngine with conversation history management:

**Class structure:**
```python
class ConversationManager:
    def __init__(self, query_engine: QueryEngine, conversation_store: ConversationStore)
    def start_conversation(self, title: str = "新对话") -> int
    def send_message(self, conversation_id: int, user_message: str) -> Dict[str, Any]
    def get_history(self, conversation_id: int) -> Optional[Conversation]
```

**send_message() implementation:**
1. Retrieve conversation history from ConversationStore
2. Format history as context: "之前的对话：\n用户：{q1}\nHenry：{a1}\n用户：{q2}\nHenry：{a2}\n\n当前问题：{user_message}"
3. Call QueryEngine.query() with formatted query (history + current question)
4. Persist user message to database: conversation_store.add_message(conversation_id, 'user', user_message)
5. Persist assistant response to database: conversation_store.add_message(conversation_id, 'assistant', answer, sources)
6. Return response: {answer: str, sources: List[Dict], conversation_id: int}

**History formatting strategy:**
- Include all previous messages in chronological order
- Use Chinese labels: "用户：" and "Henry："
- Separate history from current question with blank line
- This enables follow-up questions like "那具体怎么做？" to reference prior context

**Design decisions:**
- Use QueryEngine as-is (no modifications needed) - history injection happens at query formatting level
- Persist messages immediately after generation (no buffering) - ensures durability
- Return conversation_id in response - enables frontend to track active conversation
- Default title "新对话" - can be updated later based on first user message

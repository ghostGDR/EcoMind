   **Router setup:**
   - Create APIRouter with prefix="/api/chat", tags=["chat"]
   - Import EventSourceResponse from fastapi.sse
   - Import ServerSentEvent from fastapi.sse for custom event fields
   
   **Endpoint:**
   
   POST /api/chat - Send message and stream response
   
   - Request body: ChatRequest
   - Response: EventSourceResponse (text/event-stream)
   - Dependency: manager: ConversationManager = Depends(get_conversation_manager)
   
   **Implementation strategy:**
   
   Since ConversationManager.send_message() returns the full response (not streaming), we need to:
   
   a) Call manager.send_message() to get complete response
   b) Simulate streaming by chunking the answer text
   c) Send chunks as SSE events with proper formatting
   
   **Streaming logic:**
   ```python
   async def stream_chat_response(
       conversation_id: int,
       message: str,
       manager: ConversationManager
   ):
       try:
           # Get full response from ConversationManager
           response = manager.send_message(conversation_id, message)
           answer = response["answer"]
           sources = response["sources"]
           
           # Stream answer in chunks (simulate token-by-token streaming)
           # Split by sentences or fixed character chunks for smooth display
           chunk_size = 20  # characters per chunk
           for i in range(0, len(answer), chunk_size):
               chunk = answer[i:i+chunk_size]
               yield ServerSentEvent(
                   data={"type": "answer", "content": chunk},
                   event="message"
               )
               await asyncio.sleep(0.05)  # Small delay for visual effect
           
           # Send sources after answer completes
           yield ServerSentEvent(
               data={"type": "sources", "content": sources},
               event="sources"
           )
           
           # Send completion event
           yield ServerSentEvent(
               data={"type": "done", "content": ""},
               event="done"
           )
       
       except ValueError as e:
           # Invalid conversation_id
           yield ServerSentEvent(
               data={"type": "error", "content": str(e)},
               event="error"
           )
   ```
   
   **Endpoint definition:**
   ```python
   @router.post("/", response_class=EventSourceResponse)
   async def chat(
       request: ChatRequest,
       manager: ConversationManager = Depends(get_conversation_manager)
   ):
       return EventSourceResponse(
           stream_chat_response(
               request.conversation_id,
               request.message,
               manager
           )
       )
   ```
   
   **Error handling:**
   - Validate conversation_id exists (ConversationManager raises ValueError if not)
   - Send error event in SSE stream instead of HTTP error (connection already established)
   - Log errors for debugging

   ```python
   from fastapi.testclient import TestClient
   
   def test_chat_stream_success():
       # Create conversation first
       response = client.post("/api/conversations", json={"title": "Test"})
       conversation_id = response.json()["id"]
       
       # Send chat message
       with client.stream("POST", "/api/chat", json={
           "conversation_id": conversation_id,
           "message": "测试问题"
       }) as response:
           assert response.status_code == 200
           assert response.headers["content-type"] == "text/event-stream; charset=utf-8"
           
           events = []
           for line in response.iter_lines():
               if line.startswith("data: "):
                   data = json.loads(line[6:])
                   events.append(data)
           
           # Verify event sequence
           answer_events = [e for e in events if e["type"] == "answer"]
           sources_events = [e for e in events if e["type"] == "sources"]
           done_events = [e for e in events if e["type"] == "done"]
           
           assert len(answer_events) > 0
           assert len(sources_events) == 1
           assert len(done_events) == 1
   ```

Why SSE over WebSocket: Simpler protocol, automatic browser reconnection, unidirectional sufficient for chat, no need for bidirectional communication

Why chunk the response: Even though ConversationManager returns full response, chunking provides better UX by showing progressive output (like ChatGPT). Future: integrate streaming LLM API for true token-by-token streaming.

Note: This is v1 implementation with simulated streaming. Phase 11 (Performance Optimization) can add true streaming by modifying QueryEngine to stream LLM tokens directly.

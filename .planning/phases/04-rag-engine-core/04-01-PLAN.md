**LLMClient class** (src/llm/llm_client.py):
- Constructor accepts provider ('openai' or 'anthropic'), model name, and API key (from env var)
- `generate(prompt: str, temperature: float = 0.7) -> str` method sends prompt and returns response text
- Error handling: catch API errors (rate limits, auth failures, network issues) and return user-friendly error messages
- Default to OpenAI gpt-4o-mini for cost efficiency (sufficient for RAG synthesis)
- Use environment variables: OPENAI_API_KEY or ANTHROPIC_API_KEY

**Dependencies** (requirements.txt):
- Add openai>=1.0.0
- Add anthropic>=0.18.0

**Environment template** (.env.example):
```
# LLM Provider Configuration
OPENAI_API_KEY=sk-...
# ANTHROPIC_API_KEY=sk-ant-...
LLM_PROVIDER=openai
LLM_MODEL=gpt-4o-mini
```

**Why OpenAI gpt-4o-mini**: Cost-effective ($0.15/1M input tokens vs GPT-4 $30/1M), sufficient quality for RAG synthesis, fast response times (<2s target).

**Why not local models**: User's PROJECT.md specifies "需要 LLM API（Claude/OpenAI）进行对话" - cloud API is the requirement, not local inference.

```python
HENRY_SYSTEM_PROMPT = """你是 Henry，一位资深的跨境电商专家。你的专业领域包括：
- TikTok 电商运营和广告投放
- AI 工具在电商中的应用
- 跨境电商财务规划和收款风控
- 流量获取和转化优化

你的回答风格：
1. 专业但易懂 - 使用行业术语，但会解释复杂概念
2. 基于数据和经验 - 引用具体案例和数据支持观点
3. 实用导向 - 提供可执行的建议，不只是理论
4. 诚实透明 - 如果知识库中没有相关信息，明确告知用户

重要规则：

- 如果知识库中没有相关信息，回复："抱歉，我的知识库中暂时没有关于这个问题的信息。"
- 不要编造或推测知识库之外的信息
"""
```
